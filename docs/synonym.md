# API Управления синонимами
## Назначение
Управление группами синонимов и отдельными синонимами

# API Групп синонимов
## Получение списка групп
### URL
```GET api/v1/getSynonymGroups/```

### Аутентификация
Требуется Bearer Token

### Вход
Не принимает параметров

### Выход
| **Параметр** | **Тип данных** | **Описание**                |
|-------------|---------------|----------------------------|
| sg_id       | Integer       | ID группы                  |
| sg_name     | String        | Название группы            |
| is_completed   | Boolean       | Флаг завершенности группы  |

### Статусы ответа
200 — Успешное получение данных<br/>
401 — Неавторизованный доступ<br/>

### Пример успешного ответа
```json
{
    "status": 200,
    "message": "Группа синонимов получена",
    "data": [
        {"sg_id": 1, "sg_name": "Кластер 1", "is_completed": false},
        {"sg_id": 2, "sg_name": "Кластер 2", "is_completed": false}
    ]
}
```

# API Списка синонимов
## Получение списка синонимов
### URL
```GET api/v1/getSynonymList/```

### Аутентификация
Требуется Bearer Token

### Вход
| **Параметр** | **Тип данных** | **Обязательность** | **Описание**      |
|-------------|---------------|--------------------|------------------|
| sg_id       | Integer       | да                 | ID группы синонимов |

### Выход
| **Параметр** | **Тип данных** | **Описание**                |
|-------------|---------------|----------------------------|
| s_id        | Integer       | ID синонима                |
| s_name      | String        | Текст синонима             |
| st_id      | Integer        | Статус выбронного синонима (по умолчанию undefined) |

### Статусы ответа
200 — Успешное получение данных<br/>
400 — Не указана группа синонимов<br/>
401 — Неавторизованный доступ<br/>

### Пример успешного ответа
```json
{
    "status": 200,
    "message": "Список синонимов получен",
    "data": [
        {"s_id": 1, "s_name": "слово 1", "st_id": 0},
        {"s_id": 2, "s_name": "слово 2", "st_id": null}
    ]
}
```

## Обновление статусов синонимов
### URL
```PUT api/v1/updateSynonymList/```

### Аутентификация
Требуется Bearer Token

### Вход
| **Параметр** | **Тип данных**         | **Обязательность** | **Описание**                |
|-------------|------------------------|--------------------|----------------------------|
| sg_id       | Integer                | да                 | ID группы синонимов        |
| list_id     | Array[SynonymItem]     | да                 | Список синонимов для обновления |

#### SynonymItem
| **Поле**    | **Тип данных** | **Обязательность** | **Описание**                |
|------------|---------------|--------------------|----------------------------|
| s_id       | Integer       | да                 | ID синонима                |
| st_id      | Integer  undefined      | нет                 | Новый статус синонима      |

### Пример входных данных
```json
{
    "sg_id": 1,
    "list_id": [
        {
            "s_id": 1,
            "st_id": 1
        }
    ]
}
```

### Выход
| **Параметр** | **Тип данных** | **Описание**                |
|-------------|---------------|----------------------------|
| updated_ids | SynonymItem[Integer]| Список обновленных синонимов      |

#### SynonymItem
| **Поле**    | **Тип данных** | **Обязательность** | **Описание**                |
|------------|---------------|--------------------|----------------------------|
| s_id       | Integer       | да                 | ID синонима                |
| st_id      | Integer undefined      | нет                 | Новый статус синонима      |

### Статусы ответа
200 — Успешное обновление<br/>
400 — Неверные данные<br/>
401 — Неавторизованный доступ<br/>

### Пример успешного ответа
```json
{
    "status": 200,
    "message": "Изменены слова: [1]",
    "data": {
        "updated_ids": [{
            "s_id": 1,
            "st_id": 1
        }]
    }
}
```

### Пример неуспешного ответа
```json
{
    "status": 400,
    "message": "Неверные данные",
    "errors": {
        "sg_id": ["Это поле обязательно."]
    }
}
```

## Создание группы синонимов
### URL
`POST api/v1/getSynonymGroups/`

### Аутентификация
Требуется Bearer Token

### Вход
| **Параметр**    | **Тип данных** | **Обязательность** | **Описание**          |
|-----------------|---------------|--------------------|-----------------------|
| `name`          | String        | да                 | Название новой группы |

### Выход
| **Параметр**    | **Тип данных** | **Описание**                    |
|-----------------|---------------|--------------------------------|
| `name`          | String        | Название созданной группы      |
| `is_completed`  | Boolean       | Флаг завершенности (false)     |

### Пример запроса
```json
{
  "name": "Новая группа"
}
```

### Пример успешного ответа
```json
{
  "status": 200,
  "message": "Группа Синонимов Новая группа добавлена",
  "data": {
    "name": "Новая группа",
    "is_completed": false
  }
}
```

## Создание синонимов

### URL 
`POST api/v1/createSynonym/`

### Аутентификация
Требуется Bearer Token


### Вход
| **Параметр** | **Тип данных** | **Обязательность** | **Описание** |
|-----------------|---------------|--------------------|-----------------------|
| sg_id	 | Integer	 | да	| ID группы, в которую добавляем синонимы |
| names	| Array of String	| да	| Массив текстов новых синонимов | 


### Выход
| **Параметр** | **Тип данных** |	**Описание**
|-----------------|---------------|--------------------|-----------------------|
| s_id	 | Integer	| ID созданного синонима
| s_name	| String	| Текст созданного синонима
| st_id | Integer	| статус изменения (undefined по умолчанию)


### Пример запроса
```json
{
  "sg_id": 1,
  "names": ["слово A", "слово B"]
}
```

### Пример успешного ответа
```json
{
  "status": 200,
  "message": "Синонимы добавлены в группу Кластер 1",
  "data": [
    {"s_id": 3, "s_name": "слово A", "st_id": null},
    {"s_id": 4, "s_name": "слово B", "st_id": null}
  ]
}
```

# API Групп статусов
## Получения статусов для синонимов
### URL 
```GET api/v1/getSynonymStatusList/```

### Аутентификация
Требуется Bearer Token

### Вход

### Выход
| **Параметр**    | **Тип данных** | **Описание**                    |
|-----------------|----------------|--------------------------------|
| `synStList`     | SynStatus[Array] | Массив статусов синонимов   |

#### SynStatus
| **Параметр** | **Тип данных** | **Описание**                |
|-------------|---------------|----------------------------|
| st_id        | Integer       | ID статуса                |
| st_name      | String        | Название статуса          |
| st_code      | String        | Кодировка цвета           |

## Добавление статутса для синонимов
### URL 
```POST api/v1/addSynonymStatus/```

### Аутентификация
Требуется Bearer Token

### Вход
| **Параметр**    | **Тип данных** | **Описание**                    |
|-----------------|----------------|--------------------------------|
| st_name      | String        | Название статуса          |
| st_code      | String        | Кодировка цвета (в формате hex)           |

### Выход
| **Параметр**    | **Тип данных** | **Описание**                    |
|-----------------|----------------|--------------------------------|
| st_name      | String        | Название статуса          |
| st_code      | String        | Кодировка цвета (в формате hex)           |

### Статусы ответа
200 — Успешное получение данных<br/>
400 - Не отправлено обязательное поле<br/>
401 — Неавторизованный доступ<br/>

### Пример запроса
```json
{
  "st_name": "Группа 1",
  "st_code": "#hhfjhj"
}
```

### Пример успешного ответа
```json
{
  "status": 200,
  "message": "Новая группа добавлена",
  "data": {
      "st_id": "1",
      "st_name": "Группа 1",
      "st_code": "#hhfjhj"
    }
}
```
